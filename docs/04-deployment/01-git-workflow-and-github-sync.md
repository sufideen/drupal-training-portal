# Lesson 1 (Deployment) — Git Workflow and GitHub Sync

## What You Will Learn
- How to initialise a Git repository for your Drupal project
- What to commit and what to exclude
- How to sync your local site to GitHub continuously
- The daily developer workflow: code locally → commit → push → deploy

## Why This Matters
Git is the bridge between your local DDEV site and every deployment target
(GoDaddy, AWS, Google Cloud). Without it, you would manually upload files
every time you make a change — which breaks down immediately.

### 🔊 Auditory — Understand It

Git tracks *changes* to files, not full copies.
Every `git commit` creates a snapshot of what changed.
Every `git push` sends those snapshots to GitHub.

The `.gitignore` file tells Git which files to silently ignore.
You must ignore:
- `vendor/` — 40,000+ files generated by Composer; servers re-generate this
- `web/sites/default/files/` — uploaded user content; use cloud storage in production
- `.env` — contains passwords; never commit secrets
- `settings.php` — contains DB credentials; use environment-specific versions

The `composer.lock` file *should* be committed — it pins exact package versions
so every server uses identical PHP dependencies.

### ✋ Kinesthetic — Do It

**Step 1 — Initialise Git in Your Project**

```bash
cd ~/Documents/"Web hosting and development"/"Drupal CMS"/drupal-training-portal

git init
git branch -M main
```

**Step 2 — Create .gitignore**

```bash
cat > .gitignore << 'EOF'
# Drupal
web/sites/default/files/
web/sites/default/settings.php
web/sites/default/settings.local.php
web/sites/simpletest/

# Composer
vendor/
web/core/
web/modules/contrib/
web/themes/contrib/
web/libraries/

# DDEV
.ddev/.gitignore
.ddev/import-db/

# Environment
.env
*.env.local

# Database dumps
*.sql
*.sql.gz

# OS
.DS_Store
Thumbs.db

# Editor
.vscode/
.idea/
*.swp
EOF
```

**Step 3 — Create settings.php.example**

This is a safe, secret-free version of settings.php for the repo:

```bash
cp web/sites/default/default.settings.php web/sites/default/settings.php.example
```

Add to the bottom of `settings.php.example`:

```php
// Environment-specific settings loaded from .env or server environment
$databases['default']['default'] = [
  'database' => getenv('DB_NAME') ?: 'drupal',
  'username' => getenv('DB_USER') ?: 'drupal',
  'password' => getenv('DB_PASS') ?: '',
  'host'     => getenv('DB_HOST') ?: 'localhost',
  'port'     => getenv('DB_PORT') ?: '3306',
  'driver'   => 'mysql',
  'prefix'   => '',
];

$settings['trusted_host_patterns'] = [
  '^localhost$',
  '^127\.0\.0\.1$',
  // Add your domain:
  // '^yourdomain\.com$',
];

$settings['config_sync_directory'] = '../config/sync';
$settings['hash_salt'] = getenv('DRUPAL_HASH_SALT') ?: 'CHANGE_THIS_TO_A_RANDOM_STRING';
$settings['file_private_path'] = getenv('PRIVATE_FILES_PATH') ?: '';
```

**Step 4 — First Commit**

```bash
# Export current configuration
ddev drush config:export -y

# Stage all files
git add .

# Review what will be committed
git status

# Commit
git commit -m "Initial commit: Drupal training portal with DDEV configuration"
```

**Step 5 — Create GitHub Repository and Push**

```bash
# Create the repo on GitHub under sufideen (private by default)
gh repo create sufideen/drupal-training-portal \
  --description "Drupal 10 Training Portal — VAK-style guide for beginners and intermediates" \
  --public \
  --source=. \
  --remote=origin \
  --push
```

Your repo is now live at: `https://github.com/sufideen/drupal-training-portal`

**Step 6 — Daily Workflow Script**

Use `scripts/sync.sh` (created in the scripts folder) for your daily push:

```bash
bash scripts/sync.sh "Updated course content types"
```

## Quick Reference

```bash
git status                    # What changed?
git add .                     # Stage all changes
git add config/sync/          # Stage only config changes
git commit -m "message"       # Save a snapshot
git push origin main          # Upload to GitHub
git pull origin main          # Download latest from GitHub
git log --oneline             # View commit history
git diff                      # See exact line-level changes
```
